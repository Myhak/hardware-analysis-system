# Руководство по участию в разработке

## Git Flow

### Основные ветки
- `main` - стабильная продакшн версия
- `develop` - активная разработка

### Рабочие ветки
- `feature/название` - новая функциональность
- `bugfix/название` - исправление багов
- `release/x.x.x` - подготовка релиза
- `hotfix/название` - срочные исправления

## Процесс разработки

1. Создайте ветку от `develop`:
```bash
git checkout develop
git pull
git checkout -b feature/my-feature
```

2. Внесите изменения и закоммитьте:
```bash
git add .
git commit -m "feat: добавлена функция X"
```

3. Запустите тесты:
```bash
./scripts/run-tests.sh
```

4. Создайте Pull Request в `develop`

## Соглашения о коммитах

Используем Conventional Commits:

- `feat:` - новая функциональность
- `fix:` - исправление бага
- `docs:` - изменения в документации
- `test:` - добавление тестов
- `refactor:` - рефакторинг кода
- `perf:` - улучшение производительности
- `ci:` - изменения в CI/CD

Пример: `feat: добавлен мониторинг CPU температуры`

## Требования к коду

- Покрытие тестами ≥70%
- Соответствие стандартам кодирования
- Проходит статический анализ
- Документация для публичных API

## Тестирование

```bash
# Unit тесты
dotnet test tests/unit/

# C++ тесты
cd build && ctest

# Проверка покрытия
./scripts/generate-coverage.sh
```

## Code Review

Каждый PR требует:
- ✅ Прохождение CI/CD
- ✅ Одобрение минимум 1 ревьюера
- ✅ Отсутствие конфликтов
- ✅ Актуализированная документация
