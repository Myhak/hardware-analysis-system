# Таблицы сравнения компонентов

Этот документ содержит сравнительные таблицы различных аппаратных компонентов и технологий.

## 1. Сравнение архитектур процессоров

### x86-64 vs ARM vs RISC-V

| Характеристика | x86-64 (Intel/AMD) | ARM | RISC-V |
|----------------|-------------------|-----|---------|
| **Тип архитектуры** | CISC (сложная) | RISC (упрощённая) | RISC (упрощённая) |
| **Количество инструкций** | >1000 | ~200-500 | ~50 базовых |
| **Энергоэффективность** | Средняя (65-125W desktop) | Высокая (5-15W mobile) | Очень высокая (1-10W) |
| **Пиковая производительность** | Очень высокая | Высокая | Средняя |
| **IPC (инструкций/такт)** | 4-6 (современные) | 3-5 | 2-4 |
| **Максимальная частота** | 5.5-6.0 GHz | 3.0-3.5 GHz | 2.0-3.0 GHz |
| **Размер кристалла** | Большой (200-600 мм²) | Средний (50-150 мм²) | Малый (20-100 мм²) |
| **Векторные расширения** | AVX2, AVX-512 | NEON, SVE | V-extension |
| **Лицензирование** | Проприетарное | Лицензируемое (ARM Ltd) | Открытое (Open Source) |
| **Основное применение** | Десктопы, серверы | Мобильные, embedded | IoT, embedded, исследования |
| **Backward compatibility** | Отличная (до 8086) | Хорошая | В развитии |
| **Стоимость разработки** | Очень высокая | Высокая | Низкая-средняя |
| **Примеры процессоров** | Intel Core i9, AMD Ryzen 9 | Apple M3, Qualcomm Snapdragon | SiFive U74, Alibaba T-Head |

**Вывод:** x86-64 доминирует в высокопроизводительных вычислениях, ARM — в мобильных устройствах и энергоэффективных серверах, RISC-V — перспективная открытая архитектура для специализированных применений.

## 2. Сравнение памяти DDR4 vs DDR5 vs LPDDR5X

| Характеристика | DDR4 | DDR5 | LPDDR5X |
|----------------|------|------|---------|
| **Год появления** | 2014 | 2020 | 2022 |
| **Напряжение** | 1.2 В | 1.1 В | 0.5 В (двойное питание) |
| **Базовая частота** | 1600-3200 MT/s | 4800-8400 MT/s | 4266-8533 MT/s |
| **Пропускная способность (1 канал)** | 12.8-25.6 ГБ/с | 38.4-67.2 ГБ/с | 34.1-68.3 ГБ/с |
| **Субканалы на модуль** | 1 (64-bit) | 2 (32-bit каждый) | 2 (16-bit каждый) |
| **Банки памяти** | 16 | 32 | 16 |
| **Макс. объём модуля** | 32 ГБ (потребительский) | 64 ГБ | 32 ГБ |
| **On-Die ECC** | Нет | Да | Да |
| **PMIC** | На материнской плате | На модуле | Интегрирован |
| **Типичный CAS Latency** | CL14-CL22 | CL30-CL46 | CL28-CL40 |
| **Латентность (абс., ns)** | 9-11 ns | 9-14 ns | 8-12 ns |
| **Энергопотребление** | Базовое | -20% от DDR4 | -50% от DDR4 |
| **Основное применение** | Десктопы, серверы | Десктопы, серверы | Ноутбуки, мобильные |
| **Стоимость (за ГБ)** | $$ | $$$ | $$$$ |

**Примечание:** Латентность в тактах у DDR5 выше, но за счёт удвоенной частоты абсолютная латентность в наносекундах сопоставима с DDR4.

## 3. Сравнение накопителей: HDD vs SATA SSD vs NVMe SSD

| Характеристика | HDD | SATA SSD | NVMe SSD (PCIe 4.0) | NVMe SSD (PCIe 5.0) |
|----------------|-----|----------|---------------------|---------------------|
| **Интерфейс** | SATA III | SATA III | PCIe 4.0 x4 | PCIe 5.0 x4 |
| **Макс. пропускная способность** | 600 МБ/с | 600 МБ/с | 7000 МБ/с | 14000 МБ/с |
| **Последовательное чтение** | 120-200 МБ/с | 500-550 МБ/с | 5000-7400 МБ/с | 10000-14000 МБ/с |
| **Последовательная запись** | 100-180 МБ/с | 450-530 МБ/с | 4000-7000 МБ/с | 9000-13000 МБ/с |
| **Случайное чтение (4K)** | 0.5-1 МБ/с (100 IOPS) | 30-50 МБ/с (10K IOPS) | 80-100 МБ/с (1M IOPS) | 100-150 МБ/с (1.5M IOPS) |
| **Латентность** | 10-15 мс | 0.05-0.1 мс | 0.01-0.02 мс | 0.008-0.015 мс |
| **IOPS (4K random)** | 100-200 | 90K-100K | 700K-1M | 1M-1.5M |
| **Форм-фактор** | 3.5" / 2.5" | 2.5" | M.2 2280 / U.2 | M.2 2280 |
| **Энергопотребление** | 5-10 Вт | 2-4 Вт | 3-7 Вт | 4-9 Вт |
| **Объём (типичный)** | 1-20 ТБ | 128 ГБ - 4 ТБ | 250 ГБ - 4 ТБ | 512 ГБ - 2 ТБ |
| **Долговечность (TBW)** | Н/Д | 150-1500 ТБ | 300-3000 ТБ | 600-3600 ТБ |
| **Стоимость (за ТБ)** | $15-25 | $50-80 | $80-120 | $150-250 |
| **Надёжность (MTBF)** | 1-1.5M часов | 1.5-2M часов | 1.5-2M часов | 1.5-2M часов |
| **Использование** | Архивы, бэкапы | Общего назначения | Игры, рабочие станции | Профессиональные задачи |

**Вывод:** HDD — для массового хранения данных, SATA SSD — базовый уровень SSD, NVMe PCIe 4.0 — стандарт для современных систем, PCIe 5.0 — топовые решения с избыточной производительностью для большинства задач.

## 4. Сравнение методов оптимизации производительности

| Метод оптимизации | Сложность реализации | Прирост производительности | Применимость | Требования |
|-------------------|----------------------|----------------------------|--------------|------------|
| **DVFS (Dynamic Voltage/Frequency Scaling)** | Средняя | 15-30% энергосбережения при низкой нагрузке | Универсальная | Root-доступ, поддержка CPU |
| **NUMA-aware allocation** | Высокая | 20-40% для memory-bound задач | Серверы, HPC | Многопроцессорные системы |
| **AVX2 Vectorization** | Высокая | 2-8x для параллельных данных | Научные вычисления, ML | Поддержка CPU (2013+) |
| **AVX-512 Vectorization** | Очень высокая | 4-16x для параллельных данных | HPC, серверы | Xeon, EPYC (2017+) |
| **Cache-friendly structures** | Средняя | 10-50% для data-intensive задач | Универсальная | Понимание архитектуры |
| **Prefetching** | Средняя | 5-20% для предсказуемых паттернов | Обработка данных | Компилятор / ручная оптимизация |
| **Loop unrolling** | Низкая | 5-15% | Универсальная | Компилятор (автоматическая) |
| **Data alignment** | Низкая | 5-10% | Универсальная | alignas() в C++17 |
| **Huge pages (2MB/1GB)** | Средняя | 10-30% для больших объёмов памяти | Базы данных, HPC | Настройка ОС |
| **Multi-threading** | Высокая | Линейно до числа ядер (теоретически) | Параллельные задачи | Поддержка многопоточности |

## 6. Сравнение политик замещения кэша

### Тест: 10000 обращений, кэш на 64 блока, паттерн с локальностью

| Политика | Hit Rate | Описание | Сложность | Применение |
|----------|----------|----------|-----------|------------|
| **LRU (Least Recently Used)** | 78.4% | Вытесняет давно не использовавшиеся блоки | Средняя (O(1) с HashMap) | Общего назначения, CPU L2/L3 |
| **FIFO (First In First Out)** | 62.1% | Вытесняет первый загруженный блок | Низкая (O(1)) | Простые системы, буферы |
| **LFU (Least Frequently Used)** | 75.9% | Вытесняет редко используемые блоки | Высокая (счётчики) | Долгосрочные паттерны |
| **Random** | 58.3% | Случайное вытеснение | Очень низкая (O(1)) | Аппаратная реализация |
| **ARC (Adaptive)** | 82.1% | Адаптивная комбинация LRU+LFU | Высокая | Enterprise SSD, ZFS |
| **LRU-K** | 80.5% | Учитывает K последних обращений | Высокая | Базы данных |

**Вывод:** LRU — оптимальный компромисс между производительностью и сложностью для большинства применений.
